# Remove unnecessary Information

- Remember, every element added competes with existing elements in the design
- Unnecessary information is a distraction, and increases cognitive load
- Having a logical reason for every design element
  - Remove repeated elements
  - Avoid unnecessary words
  - Reveal less important information gradually
- Remove unnecessary style elements, colors, lines, backgrounds, animation

# Use Progressive Disclosure

  - **Progressive Disclosure**, the act of revealing information gradually as needed.
  - Show users information needed to complete a task
  - For users who want more information, provide them with easy access to more information
  - Progressive Disclosure can increase interaction cost, but can significantly reduce cognitive load speeding up decision making
    
  
# Minimalism doesn't mean Simplicity
  
- **Minimal doesn't mean simple**
- Minimal interfaces have a tendency to look clean and beautiful for users
- However, minimal interface has less elements and styles, but isn't necessarily simple
- Minimal interfaces tend to be vague and confusing, as they lack crucial information in favor of simplicity

# First Design for Small Screen

- Restricted space will force you to prioritize important information, and remove unnecessary information
- When first designing on a large screen, designers tend to want to fill space with more information
- More information can increase cognitive load

# Break-up choices to speed up decision making

- Time taken before making a decision increases with the number/complexity of choices available

## Methods to Achieve Less Choices

1. Remove choices
2. Group or categorize choices
3. Split choices into multiple steps
4. Recommend choices

### 1. Remove Choices

- Obvious way for reducing choices, remove
- Every available option on the design should earn it's place
- **If not necessary, potential distraction**

### 2. Group/Categorize Choices

- It's simpler & faster to decide between lists in a small number of categories, compared to one large list

### 3. Breakup choices into multiple steps

- Make large, complex tasks less overwhelming by breaking them up into smaller steps
- **Long forms, large & complex navigation menus** are also broken into multiple levels

### 4. Recommend choices

- Suggest popular or common choices for users, especially if a number of options are available
- If other users prefer certain choices, there is a good chance other users will too
- If there is a list of choices, show recommendations from the most common user selected options

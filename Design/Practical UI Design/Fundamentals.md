- Identify large usability risks using **Usability Testing**.
- Aim to meet **WCAG** (Web Content Accessibility Guidelines).
- **WCAG** is a benchmark for website accessibility.

# Logical Reasoning Behind Design Details

- Designing interfaces based **objective logic** instead of **subjective opinion** helps make design decisions *faster and easier*
- Having a reason behind design decisions helps articulate the decisions

# Minimize Interaction Cost

- **Interaction cost**: the sum of physical & mental effort required to achieve a task.
- Higher interaction cost means harder to achieve a task
- Interaction cost can be measured, every interaction a user performs adds up to the interaction cost of a product
- Since **interaction cost** can be measured, it can also be minimized/optimized
- More features a product has, more interaction cost it will have by default

## How to Minimize Interaction Cost

1. **Fitt's Law**: the closer and the larger a target, the faster it is to click on the target
2. **Reduce Distractions**: attention grabbing elements, like animations banners, pop-ups, and unnecessary visuals, take away attention from the user.
3. **Hick's Law**: the number of choices available to the user directly co-relate to the time it takes to make a decision
	- Reduce choices to speed up decision-making
	- Highlight small subset of items from available choices, to improve time it takes for users for decision-making

# Minimize Cognitive Load

- **Cognitive Load**: the mount of mental effort required to use an interface.
- Minimizing cognitive load will improve usability by making the product easier to use.
- Lower cognitive load will free up mental resources from the user, helping the user focus on the task they are trying to achieve

## How to reduce unnecessary cognitive load

1. **Remove unnecessary information**, decisions and styles
2. **Break-up information** into smaller groups, to clearly show relationship
3. **Use conventional patters**, the users are already familiar with
4. **Maintain consistency**, ensure elements that have similar look function in a similar way
5. **Create clear visual hierarchy**, shows level of importance of information

# Design Systems

- Design systems are pre-defined options and guidelines
- These design systems help product designers make effective design decisions
- Having endless design possibilities sounds great, in reality it hinders design decisions by making the process frustrating and time-consuming
- Design systems can be created in 3 steps
	1. Set pre-defined style options
	2. Create reusable modules
	3. Define usage guidelines

## Creating Design Systems

### Pre-defined Style Options

Having **pre-defined style options for color, text, spacing, and any style element in your product** will streamline the design process. Making the design decision process efficient for the designer and helping the team release design components faster. Pre-defined style options, aside from added design elements, refers to **text, color and spacing**.

1. **Pre-defined Color Options**
	- Define color palettes for your product
	- A color palette is small set of pre-defined colors
	- You can derive other colors in your color palette from your primary color
	- Actions, buttons, and links, are assigned to the primary color from your color palette
2. **Pre-defined Text Options**: 
	- Define a small set of typography options for different text types
	- Define font face and weight once, and reuse throughout your project
3. **Pre-defined Spacing Options**: 
	- Create a limited set of pre-defined spacing options
	- Using consistent spacing options will result in a neat, and simple user interface
4. **Pre-define Other Style Elements**:
	- Create sets of **pre-defined options** for any **style elements** you use, i.e. drop shadows, border-radius, gradient colors, etc...

### Create reusable modules

- Creating Reusable modules means having **modular design**
- **Modular design** refers to having **smaller, reusable, and replaceable parts** called **modules or components**
- Modularity in design has been used to make different types of products is an integral part of making a solid design system.
- Modularity is used to improve productivity, efficiency and consistency
#### How to implement Modularity in Design

1. Start by creating the smallest components **buttons, avatars, and form input fields**
2. Compose small components to create large, more complex components
3. Arrange components in specific layouts to create reusable page templates

# Late Binding

- Late binding is a concept in programming where result of a function or method is determined at _runtime_ rather than at _compile time_.
- Meaning the content, HTML, and the style, CSS, are separate.
- The style and content can be changed independently of each other.
- The style is applied to the content at runtime, in the browser.
- In the context of CSS, late binding refers to the ability of styles to be applied to elements based on their state or context, which can change dynamically.
- This creates a a challenge that's not encountered in other design systems.
- This also provides a powerful mechanism for styling pages. Since the styles are separate and applied at runtime the style
  - Changed and updated without needing to change content.
  - Applied to multiple pages, and elements easily.
  - Allows for dynamic styling based on user interactions, media queries, and other factors that can change the state of the page.
